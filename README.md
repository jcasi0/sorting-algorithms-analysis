# Сравнение алгоритмов сортировки

## Описание проекта

В рамках проекта проведено сравнение времени выполнения трёх алгоритмов сортировки: встроенной функции Python `sorted()`, быстрой сортировки и сортировки слиянием.
Цель - оценить производительность алгоритмов на массивах различного размера и проанализировать их поведение при увеличении объёма данных.

## Используемые технологии

- Python
- matplotlib (для визуализации результатов) 

## Методология

Для сравнения были сгенерированы массивы случайных чисел размером: 10^3, 10^4, 10^5, 10^6.
Для каждого массива измерялось время выполнения:
- встроенной функции `sorted()`
- алгоритма сортировки слиянием (merge sort)
- алгоритма быстрой сортировки (quick sort)
Результаты представлены в табличном и графическом виде.

## Результаты

| Размер массива | Python sorted() (мс) | Merge sort (мс) | Quick sort (мс) |
| -------------- | -------------------- | --------------- | --------------- |
| 1 000          | 0.079                | 1.675           | 0.792           |
| 10 000         | 0.944                | 23.140          | 10.696          |
| 100 000        | 11.047               | 279.995         | 261.237         |
| 1 000 000      | 126.467              | 3445.378        | 15709.716       |

## Выводы

- Встроенная функция `sorted()` демонстрирует наилучшую производительность на всех размерах входных данных.
- При увеличении размера массива разрыв в скорости между `sorted()` и реализованными алгоритмами существенно возрастает.
- Для массивов до 10^5 быстрая сортировка и сортировка слиянием показывают сопоставимое время выполнения.
- На массиве размером 10^6 быстрая сортировка работает значительно медленнее, что связано с особенностями выбора опорного элемента.
- Сортировка слиянием показывает более стабильное поведение на больших объёмах данных.
